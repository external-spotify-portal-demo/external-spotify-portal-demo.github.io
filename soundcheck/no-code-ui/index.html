No-code UI

The No-Code UI is a powerful user interface integrated into the Soundcheck plugin, allowing users to manage checks and tracks without the need to write YAML code. Before the No-Code UI, users had to manually write YAML code to configure new checks and tracks. Now, even non-programmers can easily add and manage checks and tracks within the Soundcheck environment.

The No-Code UI provides the same capabilities as the YAML configuration method but simplifies and visualizes the process of managing checks and tracks. Checks and tracks created through the No-Code UI are stored in the central Backstage database or a plugin-specific database, if configured.

Using the No-Code UI​

Checks​

A check involves comparing an expected outcome with the actual outcome of a process. Checks highlight whether software aligns with organizational standards and best practices.

Creating a New Check​

To create a check, follow these steps:

  1. Select the desired facts to use.
  2. Organize the facts into rules.
  3. Provide a meaningful name and description for the check.
  4. Define rules that determine whether an entity passes or fails.
  5. Combine simple expressions using "Any Of" or "All Of" for more complex rules.
  6. Optionally, add Pass/Fail messages for additional details.

Editing a Check​

Once a check is created, you can manage and edit it on its detail page. To edit a check:

  1. Navigate to the checks listing page.
  2. Locate the check you want to edit and click the edit option.
  3. Make desired changes on the details page.

Keep in mind that changes to checks can impact tracks that are already assigned with the check. Additionally, checks can be part of multiple tracks, potentially affecting different track setups.

Tracks​

Tracks consist of levels and checks, serving to align with architectural best practices and standards. Each track is made up of one or more levels, and each level contains one or more checks.

Creating a New Track​

To create a track, follow these steps:

  1. Provide a meaningful name and description for the track.
  2. Select filters and checks to include in the track.

Filters allow customization to apply tracks to specific entities in the Software Catalog, based on tags like "Java", "service", or "production".

Editing a Track​

To edit a track:

  1. Go to the tracks listing page.
  2. Locate the track you wish to edit and click the edit option.
  3. Make desired changes on the details page.

Like checks, changes to tracks are immediately reflected in anything assigned to the track.

Remember to save your changes in order for them to take effect.

Conclusion​

The No-Code UI provides an intuitive way for both programmers and non-programmers to manage checks and tracks within the Soundcheck plugin. By simplifying the process and visualizing the configuration, the No-Code UI enhances the usability of Soundcheck's capabilities.
